# Mem-T: Densifying Rewards for Long-Horizon Memory Agents

> Yanwei Yue, Boci Peng, Xuanbo Fan, Jiaxin Guo, Qiankun Li, Yan Zhang
> Peking University / NTU | 2026 | [https://github.com/yanweiyue/Mem-T](https://github.com/yanweiyue/Mem-T)

## 📄 核心贡献

- 提出 **Mem-T**：分层记忆Agent + **MoT-GRPO**（记忆操作树引导的GRPO），将稀疏终端奖励转化为密集的逐步监督
- 核心创新：**记忆操作树(MoT)**——构建多棵搜索树探索多样轨迹，通过树反向传播将奖励分配到中间节点，实现过程级信用分配
- 在LoCoMo上F1=58.65（Qwen3-4B），超越A-Mem和Mem0达14.92%，同时token使用减少24.45%

## 🔬 方法论（大白话讲解）

### 系统架构：四层记忆

Mem-T 设计了四层记忆模块：

| 记忆类型 | 内容 | 类比 |
|----------|------|------|
| **工作记忆 (Mwork)** | 会话级摘要，每步更新 | 便利贴 |
| **事实记忆 (Mfact)** | 声明性知识（谁、什么、哪里） | 百科全书 |
| **经验记忆 (Mexp)** | 程序性知识（怎么做、策略） | 操作手册 |
| **原始记忆 (Mraw)** | 原始数据存档 | 档案柜 |

每条记忆带有时间窗口 [t_start, t_end]，支持时间推理。

### 双轨决策过程

**阶段I：持续记忆构建**
- 处理输入流时，Agent主动构建新记忆
- 形成动作空间：CrtFact（创建事实）、CrtExp（创建经验）、CrtRaw（创建原始）、UpdWork（更新工作记忆）
- 演化动作空间：ADD、UPDATE、DELETE、IGNORE

**阶段II：按需记忆检索**
- 当查询到来时，Agent执行多轮检索
- 检索动作：Search(记忆类型, 查询词, topk) + Finish
- Agent自主决定检索什么、从哪个记忆库检索、何时停止

**大白话**：Mem-T 的Agent像一个图书馆管理员——持续整理新书（构建），同时在你需要时帮你找书（检索），而且能自己决定什么时候找、找哪个书架。

### MoT-GRPO：树引导的强化学习

**问题**：传统RL在长视野记忆任务中，Agent执行数百个操作后才得到一个0/1奖励 → 信用分配困难。

**解决方案：记忆操作树(MoT)**

1. **构建多棵搜索树**：对每个查询，构建G棵树（默认G=3），每棵树从根节点开始展开
2. **迭代扩展**：每轮随机选择非终端节点，生成新分支，逐步丰富搜索空间
3. **节点级奖励反向传播**：
   ```
   R(v) = 格式正确性 × (α × 证据密度 + 期望终端表现)
   ```
   - 叶子节点：表现 = F1/准确率
   - 内部节点：表现 = 子节点表现的平均值
4. **双尺度优势估计**：
   - 树内优势：同一棵树内节点的相对质量
   - 树间优势：跨所有树的全局质量比较
   - 最终优势 = 树内 + 树间

### 后顾信用分配（Hindsight Credit Assignment）

**问题**：记忆构建操作在几百步前执行，但奖励来自下游检索的结果。

**解决方案**：将下游检索轨迹的优势信号反向传播到上游构建操作

```
信用系数 = 证据对齐门(构建的内容是否是答案证据) 
         + 检索轨迹门(构建的记忆是否被检索到)
```

这样每个记忆构建操作都能获得学习信号。

### 关键设计选择

- **为什么用树而不是线性轨迹？** 树可以并行探索多条路径，更高效地分配信用
- **为什么需要双尺度优势？** 树内优势提供局部比较，树间优势提供全局竞争
- **为什么后顾信用分配？** 记忆构建的质量只有在下游检索时才能验证，需要"回顾"来评估

## 📊 实验结果

### 实验设置
- 基础模型：Qwen3-4B、Qwen3-8B
- 训练：MoT-GRPO，200步
- 数据集：LoCoMo（训练+测试）、HotpotQA、LongMemEval、NarrativeQA（OOD测试）

### LoCoMo 结果

| 方法 | 模型 | F1 | BLEU-1 |
|------|------|-----|--------|
| **Training-free** | | | |
| RAG | Qwen3-4B | 41.59 | 36.45 |
| Mem0 | Qwen3-4B | 43.71 | 36.78 |
| A-Mem | Qwen3-4B | 39.43 | 33.04 |
| GAM | gpt-4o-mini | 53.48 | 47.33 |
| **Trained** | | | |
| Memory-R1-GRPO | Mem-R1-8B | 39.25 | 31.21 |
| Mem-α | Qwen3-4B | 46.45* | - |
| **Mem-T (w/o training)** | Qwen3-4B | 49.38 | 44.11 |
| **Mem-T (GRPO)** | Qwen3-4B | 53.56 | 48.33 |
| **Mem-T (MoT-GRPO)** | **Qwen3-4B** | **58.65** | **52.63** |

- Mem-T MoT-GRPO：F1=58.65 vs GAM gpt-4o-mini 53.48（**+9.7%**）
- 即使不训练，Mem-T（49.38）也超越多数baseline

### OOD 结果

| 方法 | HotpotQA F1 | LongMemEval Acc | NarrativeQA F1 | **平均** |
|------|-------------|-----------------|----------------|----------|
| RAG | 50.13 | 56.60 | 21.17 | 42.63 |
| A-Mem | 30.46 | 61.30 | 25.18 | 38.98 |
| GAM | 52.98 | 61.80 | 28.32 | 47.70 |
| Mem-α | 58.80 | 52.00 | 28.56 | 46.45 |
| **Mem-T** | **66.35** | **65.80** | **30.29** | **54.15** |

- Mem-T 平均54.15 vs GAM 47.70（**+13.5%**）
- 在HotpotQA上优势最大：66.35 vs 58.80（+12.8%）

### 效率对比

Mem-T 在LoCoMo上每查询token使用比GAM减少约24.45%，同时性能更高。

### 消融实验

| 配置 | F1 |
|------|-----|
| 完整Mem-T | 58.65 |
| 去掉工作记忆 | 57.59 (-1.06) |
| 去掉事实记忆 | 55.25 (-3.40) |
| 去掉经验记忆 | 56.60 (-2.05) |
| 去掉原始记忆 | 56.61 (-2.04) |
| 去掉检索优化 | 53.37 (-5.28) |
| 去掉构建优化 | 55.36 (-3.29) |

- 事实记忆最关键（-3.40）
- 检索优化比构建优化更重要（-5.28 vs -3.29）

## ⚠️ 局限性

- **树构建开销**：每查询需要构建3棵树，训练时计算成本高
- **需要可验证奖励**：仅在QA等有明确答案的任务上有效
- **仅用4B模型训练**：未验证更大模型上的效果
- **记忆演化依赖时间窗口**：时间窗口选择影响性能
- **未与MemSkill等进化方法直接对比**
