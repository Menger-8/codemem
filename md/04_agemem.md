# AgeMem: Agentic Memory - Learning Unified Long-Term and Short-Term Memory Management for LLM Agents

> Yi Yu, Liuyi Yao, Yuexiang Xie, Qingquan Tan, Jiaqi Feng, Yaliang Li, Libing Wu
> Alibaba Group / Wuhan University | 2025

## 📄 核心贡献

- 提出 **AgeMem**：首个将长期记忆(LTM)和短期记忆(STM)管理**统一到同一个Agent策略**中的框架，通过工具调用让LLM自主决定何时存储/检索/更新/摘要/过滤
- 核心创新：**三阶段渐进式强化学习** + **Step-wise GRPO**——让Agent通过RL学会记忆管理策略，而非依赖启发式规则
- 在5个长视野基准上全面超越 Mem0、A-Mem、LangMem 等SOTA，同时保持更高的记忆质量

## 🔬 方法论（大白话讲解）

### 系统架构

AgeMem 的核心思路：**把记忆管理变成Agent可以调用的"工具"**，然后用强化学习让Agent学会什么时候用哪个工具。

6个记忆管理工具：

| 工具 | 目标 | 作用 |
|------|------|------|
| **ADD** | LTM | 把新知识存入长期记忆 |
| **UPDATE** | LTM | 修改已有的长期记忆 |
| **DELETE** | LTM | 删除过时的长期记忆 |
| **RETRIEVE** | STM | 从LTM检索相关记忆到当前上下文 |
| **SUMMARY** | STM | 压缩对话历史，减少token占用 |
| **FILTER** | STM | 过滤掉无关的上下文内容 |

**大白话**：以前的系统（如MemGPT）是"系统自动管理记忆"，AgeMem是"Agent自己决定怎么管理记忆"——就像从"自动挡"升级到"手动挡+自动驾驶"。

### 三阶段训练策略

AgeMem 的RL训练设计了三个递进阶段：

**阶段1：交互与存储**
- Agent与环境进行普通交互
- 学习识别什么信息值得存入长期记忆（ADD操作）

**阶段2：抗干扰与上下文管理**
- 引入干扰/无关信息
- 学习用FILTER和SUMMARY管理短期上下文
- 防止信息过载

**阶段3：综合推理与记忆协调**
- 需要同时利用LTM和STM完成任务
- 检索LTM中的知识 + 管理当前上下文 → 生成答案

**大白话**：就像训练一个学生——先学会"听课记笔记"（阶段1），再学会"排除干扰抓住重点"（阶段2），最后"综合运用知识解题"（阶段3）。

### Step-wise GRPO

标准RL的奖励只在轨迹结束时给出，但记忆操作是分散在各个时间步的。AgeMem 的解决方案：

- 将最终的任务奖励**广播到轨迹中的每个步骤**
- 这样每个记忆操作（ADD/RETRIEVE/SUMMARY等）都能获得学习信号
- 使用GRPO（Group Relative Policy Optimization）进行策略优化

### 奖励函数设计

复合奖励 = 任务表现 + 上下文质量 + 记忆质量 + 惩罚项

- **任务表现**：答案是否正确
- **上下文质量**：token使用效率
- **记忆质量**：存储的记忆是否高质量、可复用
- **惩罚**：防止上下文溢出、过度工具调用等

### 关键设计选择

- **为什么统一LTM和STM？** 以前的系统把两者分开管理（MemGPT管LTM，RAG管STM），但两者是相互影响的——好的LTM检索可以减少STM压力，好的STM管理可以提升LTM存储质量
- **为什么用RL而不是prompt工程？** 启发式规则（如"每隔N轮摘要一次"）不够灵活，RL可以让Agent学会根据任务需求动态调整策略
- **三阶段设计的意义**：渐进式训练避免了直接学习复杂协调行为的困难

## 📊 实验结果

### 实验设置
- 数据集：ALFWorld、SciWorld、PDDL、BabyAI、HotpotQA（覆盖具身操作、游戏推理、知识问答）
- 基础模型：Qwen2.5-7B-Instruct、Qwen3-4B-Instruct
- Baseline：LangMem、A-Mem、Mem0、Mem0g、No-Memory

### 关键结果

| 方法 | ALFWorld | SciWorld | PDDL | BabyAI | HotpotQA | **平均** |
|------|----------|----------|------|--------|----------|----------|
| **Qwen2.5-7B** | | | | | | |
| No-Memory | 27.16 | 13.80 | 10.15 | 50.80 | 38.36 | 28.05 |
| LangMem | 38.27 | 28.29 | 15.85 | 51.34 | 37.43 | 34.23 |
| A-Mem | 34.68 | 28.06 | 18.39 | 58.82 | 43.95 | 36.78 |
| Mem0 | 37.49 | 26.99 | 13.96 | 60.58 | 46.66 | 37.14 |
| Mem0g | 35.34 | 30.50 | 14.86 | 58.78 | 42.06 | 36.31 |
| **AgeMem** | **41.07** | **35.55** | **17.31** | **61.42** | **54.44** | **41.96** |
| **Qwen3-4B** | | | | | | |
| No-Memory | 38.51 | 47.89 | 30.14 | 55.83 | 47.48 | 43.97 |
| **AgeMem** | **48.97** | **59.48** | **35.07** | **72.56** | **55.49** | **54.31** |

- Qwen2.5-7B：AgeMem 平均41.96 vs 最佳baseline Mem0 37.14（**+13%**）
- Qwen3-4B：AgeMem 平均54.31 vs 最佳baseline A-Mem 45.74（**+18.7%**）
- HotpotQA 提升最显著：54.44 vs 46.66（+16.7%）

### 记忆质量

| 方法 | MQ (Qwen2.5-7B) | MQ (Qwen3-4B) |
|------|-----------------|---------------|
| LangMem | 0.498 | 0.465 |
| A-Mem | 0.512 | 0.587 |
| Mem0 | 0.527 | 0.543 |
| **AgeMem** | **0.533** | **0.605** |

AgeMem 存储的记忆质量最高，说明RL训练让Agent学会了更聪明地选择存储什么。

### STM管理效果

- AgeMem 的 STM 工具比 RAG 方案减少 **3-5% token使用**
- 虽然绝对值不大，但在长对话场景中累积效果显著

### 消融实验

| 配置 | ALFWorld | SciWorld | HotpotQA |
|------|----------|----------|----------|
| Base (无记忆) | 27.16 | 13.80 | 38.36 |
| +LTM工具 | 37.73 | 27.98 | 45.76 |
| +LTM+RL | 38.88 | 32.49 | 52.04 |
| **+LTM+STM+RL (完整)** | **41.07** | **35.55** | **54.44** |

- LTM工具带来最大提升（+10~14%）
- RL训练进一步提升（+6%）
- STM工具在复杂任务上贡献最大（SciWorld +3.1%）

## ⚠️ 局限性

- **需要RL训练**：不像 Mem0/A-Mem 可以直接用prompt工程实现，AgeMem 需要专门的RL训练流程，部署门槛高
- **固定工具集**：6个工具是预定义的，无法动态扩展新的记忆操作类型
- **训练数据依赖**：三阶段训练需要精心设计的训练轨迹，数据构造成本高
- **仅在5个基准验证**：未在更广泛的生产场景（如多轮客服、代码助手）中测试
- **小模型效果受限**：Qwen2.5-7B 上部分任务（如PDDL）仍不如 No-Memory baseline
