# MemSkill: Learning and Evolving Memory Skills for Self-Evolving Agents

> Haozhen Zhang, Quanyu Long, Jianzhu Bao, Tao Feng, Weizhi Zhang, Haodong Yue, Wenya Wang
> NTU / UIUC / Tsinghua University | 2026 (Preprint) | [https://github.com/ViktorAxelsen/MemSkill](https://github.com/ViktorAxelsen/MemSkill)

## 📄 核心贡献

- 提出 **MemSkill**：将记忆操作重构为**可学习、可进化的"记忆技能"**——结构化的、可复用的例程，用于从交互轨迹中提取、整合和修剪信息
- 核心创新：**闭环进化**——控制器通过RL学习选择技能，设计师定期从困难案例中进化技能库，两者交替优化
- 在LoCoMo、LongMemEval、HotpotQA、ALFWorld上全面超越SOTA，且支持**跨模型迁移**（LLaMA训练→Qwen直接用）

## 🔬 方法论（大白话讲解）

### 系统架构

MemSkill 的核心思路：**记忆操作不是固定的add/update/delete，而是可学习的"技能"**。

三个核心组件：

| 组件 | 角色 | 类比 |
|------|------|------|
| **技能库 (Skill Bank)** | 存储所有可用的记忆技能 | 工具箱 |
| **控制器 (Controller)** | 学习选择哪些技能 | 工头 |
| **执行器 (Executor)** | 用LLM执行选中的技能 | 工人 |
| **设计师 (Designer)** | 从失败案例中进化技能库 | 技术总监 |

### 技能库结构

每个技能是一个结构化的模板，包含4个部分：

```
技能名称：捕获时间上下文
(1) 目的：捕获文本中事件/活动/事实的时间上下文
(2) 何时使用：当文本提到带有时间信息的事件时
(3) 如何应用：识别关键时间元素，简洁记录
(4) 约束：只记录明确提到的时间，不推断
```

初始有4个基础技能：INSERT、UPDATE、DELETE、SKIP。随着训练推进，设计师会提出新的技能（如"捕获实体关系"、"捕获活动细节"等）。

### 处理流程

1. **分段**：将交互轨迹（如对话）分成连续的文本段（span），每段约512 token
2. **技能选择**：控制器根据当前文本段和已检索的记忆，从技能库中选择Top-K个技能（默认K=3）
3. **执行**：LLM执行器根据选中的技能，在一次调用中生成技能引导的记忆更新
4. **评估**：用训练集上的QA任务表现作为奖励，优化控制器

**大白话**：以前是"每轮对话都用固定流程处理"，MemSkill是"先看这一段内容适合用哪些技能，然后一次性处理"。类似从"流水线工人"升级到"项目经理+专业团队"。

### 闭环进化

```
训练控制器 → 收集困难案例 → 设计师分析失败 → 进化技能库 → 继续训练控制器 → ...
```

**困难案例缓冲区**：维护一个滑动窗口，记录最近的困难案例（低奖励、重复失败的查询）。

**技能进化**：
1. 聚类困难案例（KMeans），找到不同类型的失败模式
2. LLM分析每个聚类中的代表性案例，识别缺失或错误的记忆行为
3. 提出修改现有技能 + 创建新技能

**探索机制**：进化后前50步，给新技能额外的探索激励（线性衰减），帮助控制器快速适应新技能。

### 关键设计选择

- **为什么用技能而不是固定操作？** 固定操作（add/update/delete）是人类预设的，无法适应所有场景。技能是从数据中学到的，可以捕获更精细的记忆行为
- **为什么用span-level而不是turn-level？** Turn-level逐轮处理效率低，span-level可以批量处理，减少LLM调用次数
- **为什么需要技能进化？** 初始技能可能不够用，进化让技能库随数据增长而丰富

## 📊 实验结果

### 实验设置
- 基础模型：LLaMA3.3-70B-Instruct（训练）、Qwen3-Next-80B-A3B-Instruct（迁移测试）
- 数据集：LoCoMo、LongMemEval、HotpotQA、ALFWorld
- Baseline：8个（ReadAgent、MemoryBank、A-MEM、Mem0、LangMem、MemoryOS等）

### LoCoMo + LongMemEval 结果

| 方法 | LoCoMo F1 | LoCoMo L-J | LongMemEval F1 | LongMemEval L-J | **Avg L-J** |
|------|-----------|------------|----------------|-----------------|-------------|
| **LLaMA3.3-70B** | | | | | |
| A-MEM | 35.60 | 46.34 | 25.86 | 38.12 | 42.23 |
| Mem0 | 10.18 | 33.01 | 29.94 | 45.54 | 39.28 |
| MemoryOS | 38.68 | 44.59 | 14.19 | 36.50 | 40.55 |
| **MemSkill** | **38.78** | **50.96** | **31.65** | **59.41** | **55.19** |
| **Qwen3-Next-80B** | | | | | |
| A-MEM | 34.83 | 48.41 | 12.46 | 34.65 | 41.53 |
| MemoryOS | 38.55 | 44.59 | 13.26 | 36.00 | 40.30 |
| **MemSkill** | **39.28** | **52.07** | **23.75** | **59.90** | **55.99** |

- MemSkill L-J=55.19 vs 最佳baseline A-MEM 42.23（**+30.7%**）
- LongMemEval上优势最大：59.41 vs 38.12（+56%）

### ALFWorld 结果

| 方法 | ALF-Seen SR | ALF-Unseen SR | **Avg SR** |
|------|-------------|---------------|------------|
| **LLaMA3.3-70B** | | | |
| MemoryOS | 15.71 | 14.18 | 14.95 |
| LangMem | 37.86 | 35.70 | 36.47 |

- MemSkill在具身任务上优势显著：62.09 vs 最佳baseline 39.81（+56%）

### 跨模型迁移

MemSkill只在LLaMA上训练，直接迁移到Qwen，**无需重新训练**。迁移后性能仍然超越所有baseline，说明学到的技能是可复用的。

### 跨数据集迁移

LoCoMo上训练的技能库直接迁移到HotpotQA（50/100/200文档拼接），MemSkill在所有设置下都超越MemoryOS和A-MEM。

### 消融实验

| 配置 | LLaMA L-J | Qwen L-J |
|------|-----------|----------|
| w/o controller（随机技能） | 显著下降 | 显著下降 |
| w/o designer（静态4技能） | 显著下降 | 显著下降 |
| Refine-only（不新增技能） | 略有下降 | 略有下降 |
| **MemSkill（完整）** | **50.96** | **52.07** |

- 控制器和设计师缺一不可
- 新技能的引入比单纯修改现有技能更有价值

## ⚠️ 局限性

- **训练复杂度高**：需要PPO训练控制器 + LLM设计师定期进化，流程较长
- **技能库膨胀风险**：如果不控制，设计师可能持续添加低质量技能，需要回滚机制
- **依赖高质量训练数据**：困难案例的质量直接影响技能进化的方向
- **仅在4个基准验证**：未在更广泛的任务类型（如代码生成、多模态）上测试
- **LLM调用成本**：执行器每次处理span都需要LLM调用，长对话场景下成本高
