# MEM1: Learning to Synergize Memory and Reasoning for Efficient Long-Horizon Agents

> Zijian Zhou, Ao Qu, Zhaoxuan Wu, Sunghwan Kim, Alok Prakash, Daniela Rus, Jinhua Zhao, Bryan Kian Hsiang Low, Paul Pu Liang
> SMART / NUS / MIT / Yonsei University | 2025 | [https://github.com/MIT-MI/MEM1](https://github.com/MIT-MI/MEM1)

## 📄 核心贡献

- 提出 **MEM1**：端到端RL框架，让Agent在任意长的多轮任务中保持**恒定内存使用**
- 核心创新：**记忆即推理**——推理不仅提供深层洞察，还充当"工作记忆"，从收集的信息中提取关键组件
- 在16目标多跳QA任务上，MEM1-7B比Qwen2.5-14B**性能提升3.5倍，内存减少3.7倍**
- 可扩展任务构建方法：将单目标数据集组合成任意复杂的多目标任务序列

## 🔬 方法论（大白话讲解）

### 系统架构：恒定内存

传统Agent在多轮交互中不断追加所有历史到上下文 → 内存无限增长。MEM1 的解决方案：

**每个回合只保留一个压缩的内部状态（Internal State）**

```
回合t-1: <IS(t-1)> + <query(t-1)> + <info(t-1)>
    ↓ 消化吸收
回合t:   <IS(t)> + <query(t)> + <info(t)>   ← 旧的<IS(t-1)>、<query(t-1)>、<info(t-1)>被丢弃
    ↓ 消化吸收
回合t+1: <IS(t+1)> + <query(t+1)> + <info(t+1)>
```

**大白话**：就像人类做研究——你不会把每篇论文的每个字都记在脑子里，而是"消化"后只记住关键结论，然后把论文放回书架。MEM1 让Agent学会这种"消化吸收"。

### XML标签系统

MEM1 用XML标签区分不同角色：

| 标签 | 作用 |
|------|------|
| `<IS>` | 内部状态：整合记忆+推理的压缩表示 |
| `<query>` | 环境查询：向搜索引擎/工具发出的请求 |
| `<info>` | 外部观察：从环境获取的信息 |
| `<answer>` | 最终答案 |

### 关键洞察：推理即记忆

MEM1 的核心洞察：**推理过程本身就是记忆构建过程**。

在生成 `<IS>` 时，Agent需要：
1. 回顾之前的内部状态（记忆整合）
2. 处理新获取的信息（新观察）
3. 推理当前问题需要什么
4. 输出压缩的内部状态

这意味着**推理和记忆不需要分开管理**——它们在同一个表示空间中协同工作。

### RL训练：掩码轨迹

MEM1 的上下文在每个回合都会动态更新（丢弃旧的），这给RL训练带来挑战。解决方案：

**掩码轨迹（Masked Trajectory）**：
- 将多个回合的轨迹拼接成一个逻辑连贯的完整轨迹
- 使用2D注意力掩码，确保每个回合只看到当前回合的上下文
- 梯度更新只作用于Agent自己生成的token

### 多目标任务构建

现有数据集多为单目标任务（如HotpotQA的2跳QA），不足以训练长视野Agent。

MEM1 的解决方案：**组合现有数据集**

```
HotpotQA问题1 + HotpotQA问题2 → 2目标任务
HotpotQA问题1 + ... + HotpotQA问题4 → 4目标任务
... 最多16目标任务
```

Agent需要依次回答所有子问题，且每个子问题的答案可能依赖前一个。

### 奖励设计

- 使用可验证的奖励（Exact Match, F1）
- 不显式优化内存效率，但Agent自动学会管理内存作为策略的一部分
- 元信息注入：在 `<info>` 中提示"你还有X个回合"

### 关键设计选择

- **为什么用恒定内存而不是分层记忆？** 分层记忆（如MemGPT）仍然需要管理外部存储，增加复杂度。MEM1直接压缩到恒定大小的内部状态，更简洁
- **为什么推理即记忆？** 传统方法把记忆管理和推理分开（两个模块），但它们本质上是互补的——推理帮助决定什么值得记住，记忆帮助推理更深入
- **为什么用RL而不是SFT？** SFT需要专家轨迹标注，RL可以通过试错学习最优策略

## 📊 实验结果

### 实验设置
- 基础模型：Qwen2.5-7B Base
- 训练：PPO，2目标QA数据
- 评估：Wiki RAG、Online Web-QA、WebShop

### 多目标QA结果

| 模型 | 2目标 EM | 8目标 EM | 16目标 EM | 16目标 Peak Token |
|------|----------|----------|-----------|-------------------|
| Qwen2.5-14B-Inst | 0.732 | 1.55 | 0.567 | 3,840 |
| Qwen2.5-7B-Inst | 0.268 | 0.87 | 0.165 | 4,330 |
| Search-R1 | 0.452 | 0.064 | 0.009 | 2,090 |
| DeepResearcher | 0.536 | 0.73 | 0.071 | 4,890 |
| **MEM1-QA** | **0.709** | **1.87** | **1.97** | **1,040** |

- 16目标：MEM1 EM=1.97 vs Qwen2.5-14B 0.567（**3.5倍提升**）
- 16目标：MEM1 Peak=1,040 vs Qwen2.5-14B 3,840（**3.7倍减少**）
- MEM1在2目标上仅略低于14B（0.709 vs 0.732），但在16目标上远超（1.97 vs 0.567）

### 泛化能力

MEM1在2目标数据上训练，测试时泛化到**16目标**（8倍训练长度）。其他baseline在超过训练长度后性能急剧下降，MEM1保持稳定。

### WebShop 结果

| 模型 | Avg Reward | Peak Token | Inference Time |
|------|------------|------------|----------------|
| GPT-4o | 25.48 | 5,300 | N/A |
| AgentLM-7B | 63.60 | 2,240 | 3.91s |
| AgentLM-13B | 70.80 | 2,360 | 5.23s |
| **MEM1-WebShop** | **70.87** | **810** | **2.61s** |

- MEM1-7B超越AgentLM-13B（参数量的一半）
- Peak Token减少2.8倍，推理时间减少1.5倍

### 单目标QA结果

| 环境 | 系统 | EM | F1 | Peak Token |
|------|------|-----|-----|------------|
| Wiki RAG | Qwen2.5-14B-Inst | 0.422 | 0.534 | 889 |
| Wiki RAG | DeepResearcher | 0.419 | 0.503 | 1,330 |
| **Wiki RAG** | **MEM1-QA** | **0.405** | **0.471** | **563** |
| Online Web-QA | DeepResearcher | 0.372 | 0.492 | 1,027 |
| **Online Web-QA** | **MEM1-QA** | **0.397** | **0.485** | **579** |

MEM1在单目标任务上也能保持竞争力，同时内存使用大幅减少。

### 涌现行为

MEM1在训练后展现出多种涌现行为：
- **多问题管理**：同时跟踪多个子问题的进展
- **焦点转移**：当某个问题卡住时，自动切换到更容易的目标
- **交错推理**：在内部状态中将重要信息编织到决策过程中

## ⚠️ 局限性

- **需要可验证奖励**：仅在QA、数学、网页导航等有明确奖励的任务上有效，开放性任务（如创意写作）的奖励设计是挑战
- **恒定内存可能丢失信息**：对于需要精确回忆所有细节的任务，压缩可能过度
- **仅从7B模型训练**：未验证更大模型上的效果
- **依赖提示工程**：元信息注入（回合提示）和XML标签格式需要精心设计
- **未与MemSkill/Mem-α等RL方法直接对比**：不同论文使用不同评估设置
